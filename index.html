Vert Available Units API

Developer Integration Reference Guide

Overview

This private API exposes a secure, read-only endpoint designed for external systems to fetch currently available real estate units belonging to the project "Vert".

The endpoint yields a structured JSON output of active unit codes which can be fed directly into your inventory sync or CRM pipelines.

Authentication

This API requires token-style custom header credentials. Any request missing the header or using incorrect credentials will be rejected immediately with an HTTP 401 status code.

Security Credentials

Specify the username in the request headers. The credential value is: Palmier_Admin@123456

Header Key Header Value Description
username Palmier_Admin@123456 Standard Header Key (Recommended)
X-API-Username Palmier_Admin@123456 Alternative format supported by request handlers
Authorization Bearer Palmier_Admin@123456 Bearer Token standard format fallback

Request Specifications

Parameter Specification
HTTP Method GET
URL Path /api/PalmierDevelopments-Vert-Available-Units/
Content-Type application/json

Response Specification

HTTP 200 OK (Success)

Returned when authorization is successful. The payload is a JSON object listing the unit codes of all "Available" units in the "Vert" project.

{
  "available_units": [
    "FD-S1-124",
    "FD-S1-125",
    "FD-SV-145",
    "FD-SV-146"
  ]
}

HTTP 401 Unauthorized

Returned when authentication headers are invalid, misspelled, or missing.

{
  "error": "Unauthorized: Invalid or missing username header."
}

Integration Code Samples

curl -X GET "https://prometheus.com.eg/ToP3/api/PalmierDevelopments-Vert-Available-Units/" \
     -H "username: Palmier_Admin@123456"
const fetchAvailableUnits = async () => {
  const url = "https://prometheus.com.eg/ToP3/api/PalmierDevelopments-Vert-Available-Units/";
  try {
    const response = await fetch(url, {
      method: "GET",
      headers: {
        "username": "Palmier_Admin@123456",
        "Content-Type": "application/json"
      }
    });
    
    if (!response.ok) {
      throw new Error(`HTTP error! Status: ${response.status}`);
    }
    
    const data = await response.json();
    console.log("Available Units:", data.available_units);
  } catch (error) {
    console.error("Failed to retrieve units:", error);
  }
};
import requests

url = "https://prometheus.com.eg/ToP3/api/PalmierDevelopments-Vert-Available-Units/"
headers = {
    "username": "Palmier_Admin@123456",
    "Content-Type": "application/json"
}

try:
    response = requests.get(url, headers=headers)
    if response.status_code == 200:
        data = response.json()
        print("Available units:", data["available_units"])
    else:
        print(f"Error {response.status_code}: {response.json()}")
except Exception as e:
    print(f"Request failed: {e}")

Postman Testing Guide

Follow these step-by-step instructions to configure and execute requests within Postman:

  1. Create a New Request: Click on the + tab or New button and select HTTP Request.
  2. Set Method & URL: Set the HTTP request method dropdown to GET and paste the API URL: https://prometheus.com.eg/ToP3/api/PalmierDevelopments-Vert-Available-Units/.
  3. Add Header Authentication: Select the Headers sub-tab (located under the URL field). In the grid, enter:
    • Key: username
    • Value: Palmier_Admin@123456
  4. Submit Request: Click the blue Send button. Verify the status returns 200 OK and the JSON payload includes the list of available units.
  5. Test Security: Deactivate the header by unchecking the box next to username and click Send. The response status must return 401 Unauthorized.
